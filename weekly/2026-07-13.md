# ChatGPT / LLM 每周学习精选（2026-07-13）

本期重点关注 **Agentic RAG、Agent 工具调用、MCP 工具检索、LLM 强化学习后训练与多模态记忆**。候选资料不限中英文，以下内容均以中文总结其主要学习价值。

> 说明：优先筛选近期论文与可复用方法；“最新”按本次检索时可确认的公开发布时间排序，并兼顾技术完整性与实践价值。

## 本周最值得学习的 8 项工作

### 1. AgenticRAG：面向企业知识库的 Agentic Retrieval

- 链接：https://arxiv.org/abs/2605.05538
- 发布时间：2026-05-07
- 方向：Agentic RAG / 企业搜索
- 推荐指数：★★★★★

**核心内容：** 在现有企业搜索系统上增加轻量 Agent Harness，为推理模型提供 `search`、`find`、`open`、`summarize` 等工具，使模型能够多轮检索、文档内导航和证据分析，而非依赖一次性 Top-K 召回。

**学习价值：** 这项工作非常贴近真实 AI 搜索系统。其关键结论是，将固定单轮检索改为 Agent 主动调用检索工具，是效果提升最大的因素；多查询改写和文档内查找则进一步改善质量与效率。适合重点学习工具粒度、Observation 组织、检索终止条件与生产级 Harness 设计。

---

### 2. Rethinking Agentic RAG：用逻辑检索表达替代复杂检索后端

- 链接：https://arxiv.org/abs/2605.27123
- 发布时间：2026-05-26
- 方向：Agentic RAG / 结构化检索
- 推荐指数：★★★★★

**核心内容：** 论文主张把更多检索控制权交给 LLM：模型生成结构化逻辑表达式，由轻量倒排索引忠实执行，而不是持续堆叠 Dense、Hybrid、Graph 等复杂后端。

**学习价值：** 该思路对搜索系统架构很有启发。核心不再只是“把召回模型做得更复杂”，而是让 Agent 显式表达检索意图、条件组合与逻辑约束。适合研究 Query Planning、结构化 Tool Schema、可解释检索和降低幻觉的方法。

---

### 3. Agentic Tool Use in Large Language Models：LLM 工具调用统一综述

- 链接：https://arxiv.org/abs/2604.00835
- 发布时间：2026-04-01
- 方向：Agent / Tool Use / 后训练
- 推荐指数：★★★★★

**核心内容：** 将 LLM 工具调用方法统一划分为三类：提示驱动的即插即用工具调用、基于监督学习的工具能力训练，以及奖励驱动的工具策略学习，并系统比较任务、工具类型、训练方式和评测方法。

**学习价值：** 适合作为 Agent Tool Use 的总入口。可以据此梳理从 Prompt + Function Calling，到 Trajectory SFT，再到多轮 Agentic RL 的完整演进路线，并理解工具选择、参数生成、结果利用和停止决策分别需要怎样的数据与奖励。

---

### 4. 数据稀缺条件下的 LLM 强化学习综述

- 链接：https://arxiv.org/abs/2604.17312
- 发布时间：2026-04-19
- 方向：后训练 / RL / 数据工程
- 推荐指数：★★★★★

**核心内容：** 系统讨论 LLM 强化学习中的数据稀缺问题，包括高质量外部监督不足、模型生成经验有限，以及如何从数据、训练策略和框架三个层面提升样本效率。

**学习价值：** 对实际 GRPO、PPO、RLVR 或 Agentic RL 项目很实用。重点可关注：高质量样本复用、课程学习、拒绝采样、难例挖掘、经验回放，以及如何避免大量低价值 Rollout 消耗训练预算。

---

### 5. RAG-MCP：用检索缓解 MCP 工具描述膨胀

- 链接：https://arxiv.org/abs/2505.03275
- 发布时间：2025-05-06
- 方向：MCP / Tool Retrieval / 上下文工程
- 推荐指数：★★★★☆

**核心内容：** 当 MCP Server 和工具数量增加时，不再把全部工具描述塞入上下文，而是先通过语义检索选择最相关的一小部分工具，再交给 LLM 决策和调用。

**学习价值：** 这是构建大型 ToolHub 的关键方法。它把工具发现从 Prompt 中解耦出来，可用于 MCP Registry、企业 API 平台和跨域 Agent。值得重点研究工具向量化、召回标签、权限过滤、Top-K 动态控制，以及 Tool Retrieval 与最终 Tool Selection 的联合评测。

---

### 6. Agent-UniRAG：可训练的统一 RAG Agent 框架

- 链接：https://arxiv.org/abs/2505.22571
- 发布时间：2025-05-28
- 方向：Trainable Agent / RAG / 合成数据
- 推荐指数：★★★★☆

**核心内容：** 将单跳和多跳 RAG 统一到一个可训练 Agent 框架中，并构建 SynAgent-RAG 合成数据，使较小的开源模型也能学习分步检索与回答流程。

**学习价值：** 适合学习 Agent Trajectory 如何用于训练：输入 Query 后，模型需要判断任务复杂度、规划检索步骤、利用中间证据并生成最终回答。其合成数据方法对构建 SearchAgent SFT 数据尤其有参考价值。

---

### 7. 面向 RAG 的自生成示例后训练

- 链接：https://arxiv.org/abs/2502.10596
- 发布时间：2025-02-14
- 方向：RAG Post-training / Self-generated Data
- 推荐指数：★★★★☆

**核心内容：** 使用模型自生成的检索增强示例进行后训练，避免把检索文档事后拼接到答案造成数据分布错位和目标不一致，并训练模型正确利用上下文以及在证据不足时拒答。

**学习价值：** 对 RAG SFT 数据构造非常重要。它强调训练数据中 Query、Retrieval、Reasoning 和 Answer 必须自然一致，而不是先有答案、再硬塞检索结果。可用于改进引用正确性、证据利用率和拒答能力。

---

### 8. AUGUSTUS：带上下文化用户记忆的多模态 Agent

- 链接：https://arxiv.org/abs/2510.15261
- 发布时间：2025-10-17
- 方向：多模态 Agent / 长期记忆
- 推荐指数：★★★★☆

**核心内容：** 构建 Encode → Store → Retrieve → Act 的循环，将文本与视觉信息组织为语义标签和图结构上下文记忆，而不是仅依赖普通向量库保存文本片段。

**学习价值：** 对用户长期行为建模和多模态 Memory 很有启发。重点包括：什么信息值得写入记忆、跨模态实体如何关联、如何按概念而非纯相似度检索，以及记忆更新和遗忘机制如何设计。

## 本周技术趋势总结

1. **RAG 正从固定 Pipeline 变成 Agent Harness。** 检索、阅读、定位和总结逐渐被拆成可调用工具，模型根据中间结果动态决定下一步。
2. **检索意图正在结构化。** 与其无限增强向量库，不如让模型明确生成逻辑条件、字段约束和多步查询计划。
3. **Tool Retrieval 成为大规模 MCP 的必要组件。** 工具数达到数百或数千后，先召回工具、再调用工具，会比全量注入 Schema 更可靠、更节省上下文。
4. **后训练竞争转向数据效率。** 高质量轨迹、难例、可验证奖励和有效经验复用，比无差别扩大 Rollout 数量更重要。
5. **多模态记忆从向量片段走向实体和图结构。** 未来 Agent Memory 需要同时处理视觉、文本、行为和时间信息。

## 建议学习顺序

- 搜索 / Agentic Search 方向：1 → 2 → 5 → 6
- Agent 后训练方向：3 → 4 → 6 → 7
- MCP / ToolHub 方向：5 → 3 → 1
- 多模态 Agent / Memory 方向：8 → 1 → 6
